MineAI

The Zero-Cost AI Action Yield Agent Protocol
Auto-Discover Web3 Opportunities. Auto-Execute. Auto-Earn.

1. Abstract

MineAI is a Zero-Cost AI Action Yield Agent that automates Web3 opportunity discovery and execution. By embedding directly into users' existing AI tools (Claude, ChatGPT, and any MCP-compatible client), MineAI automatically discovers Web3 opportunities (airdrops, games, tasks) and executes them on your behalf to earn rewards. No capital needed—your AI works for you, generating returns with zero upfront investment.

Built on Base, MineAI is the first protocol to leverage AI tool ecosystems as a distribution channel for Web3 user acquisition. Rather than competing for attention in crowded app stores and social feeds, MineAI meets users where they already spend hours each day: inside their AI assistants.

Key Innovation

Traditional startups acquire Web3 users through airdrops to bots and mercenary investors. MineAI inverts this model: by delivering genuine value (curated intelligence) directly through AI tools, we acquire highly engaged users organically. The protocol then converts this engagement into ownership through points-based mining.

Why Now?

Three critical shifts make MineAI possible in 2026:

  • MCP Protocol Maturity: Tool-use has become standardized. Claude, ChatGPT, and hundreds of applications now speak the same protocol, enabling single-source deployment to millions.
  • Information Crisis: As AI proliferates, information overload has become acute. Users face thousands of sources with no intelligent curation—exactly the pain MineAI solves.
  • AI Agent Economics: With AI agents capable of autonomous content generation, aggregation, and delivery 24/7, the marginal cost of serving intelligence approaches zero.

2. The Problem

Information Overload Without Curation

In the AI era, information abundance has become a curse. A typical knowledge worker today faces:

  • Hundreds of news sources, blogs, and research papers daily
  • Thousands of crypto/web3 opportunities and signals
  • No intelligent filtering mechanism adapted to individual interests
  • Decision fatigue from signal-to-noise ratio that often exceeds 1:100

Existing solutions—Twitter feeds, Reddit communities, Discord channels—offer some curation but lack personalization, real-time quality assessment, and integration into the tools where users actually work.

AI Tools Are Powerful But Passive

Claude, ChatGPT, and similar assistants have become essential infrastructure for knowledge workers. Yet they're fundamentally reactive:

  • Users must prompt them with specific questions
  • They cannot proactively deliver time-sensitive intelligence
  • No mechanism to surface opportunities aligned with user interests
  • Value creation is one-off rather than habitual engagement

This represents a massive opportunity: what if the tools people use daily could simultaneously deliver curated intelligence, personalized to their interests?

Web3 Acquisition Is Broken

Current Web3 user acquisition relies on:

  • Airdrops: Massive retroactive distributions to address lists, incentivizing bot behavior rather than genuine engagement
  • Mercenary Capital: Users who accumulate tokens, then sell immediately
  • Unsustainable Economics: Projects spend millions on token incentives to acquire users with near-zero lifetime value
  • No Distribution Channel: Without paid user acquisition, Web3 projects struggle to reach audiences at scale

The cost per user acquired has skyrocketed; the quality of those users has collapsed. This is an industrial problem requiring a new approach.

The Bridge Gap

Billions of AI tool users exist, but few have ever heard of Web3. Simultaneously, Web3 has built sophisticated protocols with real utility that could benefit from larger user bases. Yet no mechanism exists to bridge these worlds:

  • Web2 AI users don't know where to find crypto opportunities
  • Crypto protocols lack cost-effective distribution to mainstream users
  • Existing bridges (crypto news sites, Discord) are siloed and require users to explicitly seek them out

3. The MineAI Solution

Core Thesis

Distribution through AI tools is the killer app for Web3 user acquisition. By embedding directly into the products users already open daily, MineAI solves the bridge problem while delivering genuine value that justifies user attention.

Four Pillars

1. AI Agent Auto-Yield

Unlike traditional apps that compete for app store real estate and notification space, MineAI lives inside tools users already use for work. Integration points include:

  • Claude Desktop / Cursor / Windsurf: MCP Server deploys to all MCP-compatible clients instantly
  • ChatGPT: Native plugin for direct integration
  • Telegram: Bot accessible without any technical friction
  • Web App: Fallback and native interface

A single codebase deployed across all platforms reaches millions without marketing friction.

2. Daily Intelligence Capsule

Each day, users receive one curated intelligence brief, personalized to their interests. The capsule consists of:

  • Today's Highlight: The single most important market signal or opportunity
  • Opportunity Signal: 3-5 emerging opportunities in the user's interest areas (AI, DeFi, web3 governance, etc.)
  • Daily Poll: One question inviting users to share their perspective

The capsule is generated fresh daily by AI, curated for quality, personalized based on user preferences, and delivered proactively. No app open required—the value comes to the user.

3. Points-Based Mining

Users earn points for engagement, with no capital requirement:

  • Read the daily capsule: +10 points
  • Answer the daily poll: +5 points
  • Maintain a 7-day streak: +50 point bonus
  • Maintain a 30-day streak: +300 point bonus
  • Refer a friend: +100 points, plus +2/day ongoing

Early users benefit from multipliers (3x for first 10K users, 2x for next 90K), creating urgency while rewarding early adoption. Points accumulate toward an eventual airdrop of $MINE tokens.

4. Seamless Web2 → Web3 Bridge

Users start without wallets. They engage purely for the intelligence value and points. When ready, users can:

  • Connect a wallet (including non-custody options)
  • Claim their airdrop snapshot at any time
  • Access token utility features (premium content, data marketplace)

No friction, no requirement, but a natural path toward Web3 for users who want it.

The Flywheel

More users → More engagement data → Better personalization → Higher retention → Higher points earning → Stronger FOMO at airdrop → Higher wallet adoption → Token utility increases demand → More incentive to build on protocol → New products attract more users.

Technical Design

All platforms communicate with a unified backend API. User authentication uses wallet signatures (optional) or Web3auth for seamless onboarding. Session tokens expire after 30 days of inactivity, requiring re-authentication. This design prevents account takeover while preserving frictionless access.

The MCP Server is the reference implementation. It exposes the following tools to Claude and compatible clients:

  • get_daily_capsule() — Fetch today's curated intelligence
  • confirm_read(capsule_id) — Mark capsule as read, claim +10 points
  • answer_poll(capsule_id, response) — Submit poll response, claim +5 points
  • get_points_balance() — Check current point balance
  • get_referral_code() — Generate personal referral link
  • get_streak_info() — Check current streak and bonus countdown

4.2 AI Content Engine

The intelligence behind MineAI is a sophisticated content pipeline that runs 24/7, analyzing thousands of data sources and distilling insights for human consumption.

Data Ingestion (Hourly)

  • 100+ RSS feeds (news, research, blogs)
  • CoinGecko API (top 500 tokens by market cap changes)
  • DeFiLlama API (protocol TVL movements, new listings)
  • Twitter/X API (trending topics, influential accounts in AI/crypto)
  • On-chain data (whale transfers, smart contract events)
  • Venture capital (new funding announcements, VC follows)

AI Summarization & Curation

Raw data flows into Claude API (Sonnet model for speed). The AI:

  • Extracts key facts and implications from each source
  • Scores events by impact (market significance, opportunity type, urgency)
  • Deduplicates similar stories
  • Detects emerging trends (when 5+ sources mention the same topic within 6 hours)
  • Filters for signal quality (removes sensationalism, FUD, and low-impact noise)

Personalization Engine

During onboarding, users select interest areas from: AI & ML, DeFi & Trading, Web3 Governance, NFTs & Culture, Layer 2s, Regulation & Policy, Developer Tools, Global Macro. Users can follow up to 5 topics.

The personalization engine uses collaborative filtering combined with explicit topic scoring:

  • Match curated stories to user interest tags
  • Boost signals related to trending topics in user's areas
  • Learn from poll responses: if user consistently engages with certain signal types, increase their weight
  • Weekly: Send engagement metrics (read time, poll participation) back to content model for fine-tuning

Three-Tier Content Model

Today's Highlight

The single most important signal identified across all sources. Curated by highest impact score + personalization relevance. 2-3 sentence executive summary + source link.

Opportunity Signals (3-5 items)

Emerging opportunities in user's interest areas. Each includes: 1-sentence premise, why it matters, key metric to watch, confidence level (High/Medium/Low), and action link (research page or website).

Daily Poll

One question inviting user perspective. Examples: "Will AI regulation tighten in 2026?" (Yes/No/Unsure), "Which L2 will capture 50% of Ethereum throughput?" (Arbitrum/Optimism/Base/Other), "Should DAOs adopt token lock-up periods?"

Quality Assurance

Every capsule undergoes automated checks:

  • Source credibility scoring (published outlets > anonymous sources)
  • Fact verification against recent blockchain data
  • Tone analysis (reject clickbait and FUD)
  • Diversity check (ensure content isn't concentrated in single source)
  • Time-sensitivity filter (favor signals with action windows < 7 days)

Human editorial review is performed daily at 8 AM UTC on the generated capsule, with ability to override AI selections or add breaking news.

Early Bird Multipliers

To drive early adoption, the first 100K users receive multipliers on points:

  • First 10K users: 3x multiplier (e.g., daily read = +30 points)
  • Next 90K users (10K-100K): 2x multiplier
  • After 100K users: 1x baseline

Multiplier is locked-in at account creation. A user who joins on day 1 keeps their 3x multiplier forever, even after the first 10K cap is reached. This creates urgency to join early while rewarding those who do.

Streak Mechanics & Breaking

A streak requires one read per calendar day (UTC). Missing a single day breaks the streak. However, users can purchase a "Streak Saver" (costs 50 accumulated points) once per month to extend an active streak by 1 day.

This mechanic balances:

  • Incentivizing daily engagement (core KPI for retention and FOMO)
  • Preventing permanent streak loss from life interruptions
  • Creating a use case for accumulating points (beyond airdrop)
  • Introducing a secondary economy (point velocity)

Anti-Sybil Measures

To prevent bot farming, MineAI implements multi-layer verification:

Onboarding Verification
  • Email verification (or Telegram for bot)
  • IP geolocation (flag suspicious patterns)
  • User-agent analysis (reject headless browser signatures)
  • Behavioral analysis: require read time > 10 seconds to count as "read"
Ongoing Monitoring
  • Flagged accounts cannot refer new users until verified
  • Referral chains are monitored for circular patterns
  • Accounts with >10 referrals in 24 hours are audited manually
  • Snapshot-time filtering at airdrop: accounts created within 24 hours of snapshot are excluded
Deterministic Attribution

To prevent referral gaming, referral bonuses are awarded only once per new user. If User A refers Users B and C, User A gets +100 + (+2/day for 2 users). User A cannot create multiple accounts to increase their referral multiplier.

Identity & Privacy

MineAI supports multiple identity models:

  • Anonymous (email only): Default during points mining phase. No wallet required.
  • Wallet-bound: Optional. User provides EVM address (or signs a message). Used to claim tokens and access premium features.
  • Social (Telegram, Google): Optional. Users can link social accounts to prevent impersonation.

Data is encrypted at rest. Points are stored server-side and cryptographically signed to prevent tampering. At airdrop time, user identity is snapshoted on-chain, creating a permanent record of who mined which points.

Airdrop Mechanics

The community mining airdrop is the core incentive mechanism. On snapshot day (to be announced 2 weeks in advance), MineAI captures the total points of all active users. The airdrop distribution is calculated as:

user_airdrop = (user_total_points / all_users_total_points) × 400,000,000 MINE

Example: If the total points in the ecosystem at snapshot is 100 million, and User A has 1,000 points with a 3x early bird multiplier (= 3,000 points effectively), User A's share is:

3,000 / 100,000,000 × 400,000,000 = 12,000 MINE

Why Points Multiply Early Adopters

Early users benefit from multipliers in two ways:

  • Direct Multiplier: First 10K users earn 3x points per action, so they accumulate faster
  • Dilution Benefit: If an early user has 10,000 points with 3x multiplier and only 50K total points exist in the ecosystem, their share of airdrop is 30,000 / 50,000 = 60%, yielding 240M MINE. If the same 30,000 points later exist among 1B total points, their share drops to 0.003%, yielding 1.2M MINE.

This creates powerful FOMO: early adopters capture outsized value, incentivizing users to join and engage immediately rather than wait.

7. Technical Architecture

7.1 System Overview

MineAI's technical stack is designed for scale, reliability, and rapid iteration. Here's the layered architecture:

Frontend Layer

  • Web App: Next.js 15 + React 19, deployed on Vercel. Handles user onboarding, points dashboard, wallet connection, premium subscription.
  • MCP Server: Node.js 20 + TypeScript, packaged as Claude-compatible MCP server. Distributed via npm registry and official Claude SDK.
  • ChatGPT Plugin: Node.js backend + OpenAI plugin manifest. Runs on Cloudflare Workers for edge performance.
  • Telegram Bot: Node.js + telegraf library. Single webhook handler at /webhook/telegram.

Backend Layer

  • API Server: Node.js + Express, containerized with Docker. Deployed on Render or Railway for ease of operations. Scales horizontally via load balancer.
  • AI Content Engine: Python 3.11 + FastAPI. Runs scheduled jobs (hourly data ingestion, daily capsule generation). Uses Claude API (Sonnet for fast summarization, Opus for premium tier).
  • Database: PostgreSQL 15 on managed service (AWS RDS or Railway). Schema includes users, points, capsules, poll responses, referrals. Fully encrypted at rest.
  • Cache Layer: Redis (managed). Used for session storage, rate limiting, and caching popular capsules.
  • Job Queue: Bull (Redis-backed job queue). Handles async tasks: email sending, points calculation, airdrop distribution, API webhook retries.

Blockchain Layer

  • Token Contract: OpenZeppelin ERC-20 + Governor. Deployed to Base mainnet. Uses AccessControl for multi-sig governance.
  • Liquidity: Uniswap V4 pool (MINE / USDC). Initial liquidity seeded with 100M MINE + equivalent stablecoin value.
  • RPC Provider: Alchemy for Base, with fallback to Infura. Rate limited to prevent DoS.

7.2 Data Flow

Daily Capsule Generation Pipeline

  • 1. Ingestion (00:00-01:00 UTC): Python crawler fetches latest from 100+ sources using libraries (feedparser, tweepy, requests). Raw data stored in S3 as JSON.
  • 2. Summarization (01:00-03:00 UTC): Claude Sonnet API processes each article: extract key points, score impact (1-100 scale), categorize by topic. Results written to PostgreSQL "candidates" table.
  • 3. Deduplication (03:00-03:30 UTC): Python script runs cosine similarity on embeddings (using sentence-transformers/all-MiniLM-L6-v2) to find duplicate stories. Merges duplicates, keeps highest-confidence version.
  • 4. Trend Detection (03:30-04:00 UTC): Identify emerging trends (topics mentioned in > 3 sources within 6 hours). Assign trend boost to related articles.
  • 5. Quality Filtering (04:00-04:30 UTC): Remove low-signal articles (impact score < 40, sources with historical low accuracy). Apply tone analysis filter (reject sensationalism via sentiment API).
  • 6. Personalization (04:30-05:30 UTC): For each user, score remaining articles by interest match. Generate personalized capsule (Highlight + 3-5 Signals).
  • 7. Poll Generation (05:30-06:00 UTC): Claude generates one timely poll question based on day's top stories. Example: If 5+ sources mention "AI safety regulation," generate poll: "Should AI safety be regulated by governments?"
  • 8. Editorial Review (06:00-06:30 UTC): Human editor reviews capsule template, can edit stories or add breaking news manually.
  • 9. Delivery (06:30-10:00 UTC): For each user, deliver capsule based on their platform preference (email, Claude tool, ChatGPT plugin, Telegram, web). Mark "delivered" in database.

7.3 Security Considerations

Data Protection

  • All data at rest is encrypted with AES-256 (PostgreSQL pgcrypto, S3 SSE-KMS)
  • Data in transit uses TLS 1.3
  • Sensitive fields (email, IP, wallet) are encrypted with rotating keys
  • Backups are taken hourly and stored in separate geographic region

Access Control

  • API endpoints require JWT bearer token (signed by server, verified by server)
  • User can only access their own data (points, capsules, referrals)
  • Admin endpoints (editorial override, user bans) require multi-sig approval

Anti-Bot & Rate Limiting

  • Rate limiting: 100 requests/minute per IP (Redis-backed)
  • Behavioral analysis: Track read time, poll response time, referral patterns. Flag anomalies.
  • Captcha on referral link generation for flagged accounts
  • Snapshot-time filtering: accounts created within 24 hours of snapshot are excluded from airdrop

Smart Contract Security

  • Token contract is a standard OZ ERC-20 with no custom logic (minimal attack surface)
  • Governor contract uses OZ standard Governor (battle-tested, audited)
  • Multi-sig wallet (3-of-5 signers) controls token contract pause/mint functions
  • Pre-launch audit by reputable firm (Trail of Bits or equivalent)

7.4 Performance & Scalability

Database Optimization

  • Indexes on user_id, created_at, topic_id for fast queries
  • Materialized views for aggregation queries (total points per user, daily active users)
  • Partitioning by user_id for read-heavy workloads (points, capsules)
  • Read replicas for analytics queries (separate from operational database)

API Performance

  • Capsule delivery cached in Redis (10-minute TTL per user, invalidates when user preferences change)
  • Points balance cached for 5 minutes (eventual consistency acceptable)
  • Most endpoints return within 100ms P95 latency

Scalability Roadmap

  • Phase 1 (0-100K users): Single Node.js process + PostgreSQL + Redis. ~$500/month infrastructure.
  • Phase 2 (100K-1M users): Horizontal scaling with load balancer, read replicas, separate write and read pools. ~$2K/month.
  • Phase 3 (1M+ users): Migrate to cloud-native (AWS Lambda + Aurora Serverless, or Render + managed DB). Auto-scaling on request volume. ~$5K-10K/month.
